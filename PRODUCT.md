# Product Context: ShaderBuild

## Register
product

## Users
- Product Designers: Creating animated hero backgrounds, section dividers, bento glows, glass panels, and text fills.
- Frontend Developers: Seeking clean GLSL, CSS, and React-friendly export paths.
- Creative Technologists: Building reusable shader systems.
- Design Teams: Standardizing brand motion, palettes, filters, and section visuals.

## Brand Values
- Precision-engineered: Visual parameters map directly to real technical properties, offering both designer-safe sliders and developer-level code access.
- Dark creative studio: Emulates high-end creative environments like Figma, Spline, or Cinema 4D. No bright distractions.
- Performance-focused: Smooth animations, fast render loops, and efficient code generation.

## Tone
- Professional, technical, and direct.
- Restrained and utilitarian.
- Focused on output quality.

## Strategic Principles
- Code is an expert lane: The user can edit shader code directly, but the primary interactions happen through visual parameters.
- WebGL and CSS are peers: CSS-native shader-style visual recipes (gradients, blend modes, masks, and SVG filters) are treated as first-class citizens, not secondary alternatives to GLSL.
- High fidelity previews: The preview canvas is the center of gravity. It is the largest element, running at high FPS.
- Zero-clutter MVP: No graph editors, layers, timelines, or community feed tabs. The editor focuses strictly on the active canvas.

## Anti-References
- Generic SaaS-cream layouts: White backgrounds with blue CTA buttons.
- Over-designed AI glows: Bright gradients on every border, button, and input card. Glow should be limited to the preview art itself.
- Jane Doe dummy data: Avoid generic names in lists, code variables, or exports. Use realistic technical presets.
